HTML Maquetación contenedores.

La maquetación en HTML es el proceso de organizar y estructurar los elementos de una página web para que el navegador sepa qué es cada parte y cómo debe distribuirse. Actualmente, el estándar es la Maquetación Semántica, que consiste en usar etiquetas que describen el significado del contenido, no solo su apariencia.

Antes se usaban div para todo. Hoy, usamos etiquetas que le dicen a Google y a los lectores de pantalla exactamente qué están procesando.

header: El encabezado. Suele contener el logo y el título.
main: El contenido principal de la página (solo debe haber uno por documento).
section: Agrupa contenido relacionado por un tema.
article: Un bloque de contenido independiente (como un post o una noticia).
aside: Contenido lateral o complementario (publicidad, widgets).
footer: El pie de página (copyright, contacto, redes sociales).

El Diseño y Posicionamiento con CSS

El diseño y posicionamiento con CSS es fundamental para crear páginas web atractivas y funcionales. CSS (Cascading Style Sheets) permite controlar la apariencia de los elementos HTML, incluyendo colores, fuentes, tamaños y, lo más importante, la disposición de los elementos en la página. Con CSS, puedes usar técnicas como Flexbox y Grid para organizar el contenido de manera flexible y responsiva, adaptándose a diferentes tamaños de pantalla. Además, CSS ofrece propiedades para controlar el posicionamiento de los elementos, como position (static, relative, absolute, fixed) y z-index para manejar la superposición. Un buen diseño con CSS no solo mejora la estética de la página, sino que también contribuye a una mejor experiencia de usuario al facilitar la navegación y la legibilidad del contenido.

Todo elemento en HTML es visualizado como una caja rectangular compuesta por cuatro capas fundamentales:
Content: El contenido real (texto, imágenes).
Padding: El espacio interior entre el contenido y el borde.
Border: La línea que rodea al elemento.
Margin: El espacio situado fuera del borde que separa al elemento de los demás
.

Selectores y Estados

Para aplicar estilos, se usan selectores como etiquetas, clases (.menu), IDs (#titulo) o el selector universal (*). Además, existen pseudoclases para definir estados (como :hover para el mouse) y pseudoelementos para partes específicas (como ::before o ::after).

JavaScript

Colaboración Backend/Frontend: Java se encarga de la lógica y los datos (Backend), mientras que HTML/CSS gestionan la estructura y el diseño (Frontend).
Spring Boot: Es el estándar de la industria utilizado para crear aplicaciones web con Java de forma sencilla.
Thymeleaf: Es el "motor de plantillas" que utiliza Java para procesar peticiones y enviar información dinámica al HTML.
Comunicación: Java recibe la visita del usuario a través de un controlador, prepara los datos y busca un archivo de vista (como indice.html) para mostrar la información.

Fundamentos del Lenguaje Java

Tipos de Datos: Java requiere que siempre se declare el tipo de dato:
Primitivos: int (enteros), double (decimales), boolean (lógica), char (carácter único).
Objetos: String para cadenas de texto

Entrada y Salida:
Salida: System.out.println() imprime en consola con salto de línea.
Entrada: Se utiliza la clase Scanner (import java.util.Scanner) para leer datos del teclado como nextLine() o nextInt()

Estructuras de Control:Condicionales: if y else deciden el camino del programa según una condición.
Bucles: for se usa cuando se conoce el número de repeticiones; while se ejecuta mientras una condición sea cierta.
